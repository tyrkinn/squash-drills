# Squash Drills

Статическое мобильное приложение: библиотека упражнений по сквошу и словарь терминов. Этот файл фиксирует язык предметной области — что каким словом называется и где границы между близкими понятиями.

## Language

**Дрилл**:
Одно упражнение по сквошу — набор инструкций (описание, шаги, подсказки) и схема корта, для соло или парной отработки одного навыка.
_Avoid_: Тренировка (это сессия из нескольких дриллов, не одно упражнение), exercise.

**Термин**:
Запись словаря — понятие сквоша (удар, зона корта, правило или тактический принцип) с коротким определением (`ru`/`en`/`def`). Принадлежит одной из категорий: удары, корт, правила, тактика.
_Avoid_: Слово, понятие.

**Ссылка на термин**:
Упоминание Термина внутри прозы Дрилла, помеченное автором вручную и показанное читателю как тапабельный элемент, открывающий определение этого Термина. Не автоопределяется по тексту — ставится явно при первом осмысленном вхождении термина в дрилле.
_Avoid_: Подсветка, автолинк (связь не выводится эвристикой из текста — она задана в данных).

### Язык движка схем

**Схема корта**:
Вид корта сверху (передняя стена вверху) с нанесёнными позициями и действиями. Описывает один дрилл; может проигрываться как анимация.
_Avoid_: Диаграмма (ок как синоним, но канон — «схема»), картинка.

**Scene**:
Нормализованная модель одной схемы — `court`, `entities`, `steps`, `zones`. Единая промежуточная форма, в которую компилируются и DSL, и старый объект-формат диаграмм. Рендерер и плеер знают только про Scene.

**Entity**:
Действующее лицо на схеме — игрок (основной `A` / партнёр `B`). Имеет позицию и может двигаться между шагами.
_Avoid_: Объект, актёр.

**Step (шаг)**:
Группа действий, которые происходят **одновременно**. Длительность шага = максимум длительностей его действий. Плеер перематывает анимацию по шагам.
_Avoid_: Кадр, фаза.

**Action (действие)**:
Элементарное событие внутри шага: удар (drive/cross/boast/…), перемещение игрока (`move`) или пауза (`wait`). У действия есть длительность (по умолчанию — по типу).
_Avoid_: Удар (это лишь один из видов действия), движение.

**Zone (зона)**:
Подсвеченная прямоугольная область корта — цель или игровая зона. Не действие, не анимируется.

**Playhead**:
Текущая позиция воспроизведения во времени сцены. Плеер двигает playhead; рендерер по нему рисует кадр.
_Avoid_: Курсор, таймкод.

## Flagged ambiguities

- **Дрилл vs Тренировка**: «дрилл» — это одно упражнение; «тренировка» зарезервирована за будущим понятием сессии из нескольких дриллов. Не использовать «тренировка» как синоним «дрилла».
- **Action vs Удар**: «удар» (drive, boast…) — лишь один из видов Action; перемещение игрока и пауза — тоже Action. Не называть все действия «ударами».

## Example dialogue

— В описании дрилла встречается слово «никкер» — оно станет ссылкой на термин?
— Да, если автор пометит его при первом вхождении. Сам по себе текст не сканируется: связь — это ссылка на термин, заданная в данных, а не автолинк. Падеж слова в тексте любой («никкером»), но ссылка ведёт на Термин `nick` в именительном падеже.
— А если дрилл упоминает «никкер» трижды?
— Ссылку ставим только на первое вхождение в дрилле — дальше читатель уже знает слово.
