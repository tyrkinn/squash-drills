# Движок схем корта рендерит в SVG, а не Canvas/WebGL

Встраиваемый движок схем сквош-корта (статика, позиции игроков, удары, зоны, анимация) рендерит в **SVG**. Canvas 2D и WebGL рассмотрены и отклонены.

Схема корта — это 2D-чертёж с парой движущихся объектов (1–2 игрока, мяч, несколько стрелок) и текстовыми подписями, который должен быть чётким на любом размере, темизируемым под CSS-токены проекта, доступным (`role=img`), интерактивным (подсветка/клик/возможный редактор) и работать офлайн без сборки и зависимостей. SVG даёт всё это из коробки: каждый элемент — адресуемый DOM-узел, поэтому хит-тестинг, hover-подсветка (чистый CSS) и события — бесплатно; текст и линии чёткие при масштабировании; темизация через CSS-переменные.

## Considered Options

- **Canvas 2D** — плавно для множества объектов, но теряет чёткость текста при масштабе, нет CSS-темизации и a11y, хит-тестинг (какой объект под курсором) пришлось бы считать вручную. Избыточно для пары движущихся объектов.
- **WebGL** — оправдан для 3D или тысяч спрайтов: шейдеры, матрицы, нет нативного текста, нет a11y, кратно больше кода. Оверкилл для линейной схемы.

## Consequences

- Интерактив (hover-подсветка зон, клик по элементам, будущий редактор «поставь игрока / протяни удар») реалистичен именно потому, что SVG-элементы адресуемы и pointer-события дают координаты.
- Слой рендерера абстрагирован за интерфейсом `(scene, t) → DOM`: DSL-парсер и плеер от него не зависят, поэтому canvas/webgl-бэкенд можно подключить позже, не переписывая остальное.
- Анимация немногих элементов — через `requestAnimationFrame`/CSS; для линейной схемы этого достаточно.
