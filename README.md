# Squash Drills

Мобильное веб-приложение со 24 упражнениями по сквошу и словарём из 37 терминов. Для соло и парных тренировок. Полностью статическое — без сервера, без бэкенда, без сборки. Можно открыть `index.html` локально или задеплоить одним перетаскиванием папки.

## Что внутри

```
squash-drills/
├── index.html          # SPA, одна страница, hash-роутинг
├── app.css             # стили (светлая тема)
├── app.js              # логика: фильтры, рендеринг, таймер, избранное
├── drills.js           # библиотека из 24 упражнений
├── glossary.js         # словарь терминов (удары, корт, правила, тактика)
├── court.js            # движок схем корта (inline SVG)
├── manifest.json       # PWA — устанавливается на домашний экран
├── sw.js               # service worker — работает офлайн
└── icons/              # иконки PWA
```

## Особенности

- **24 упражнения**: 12 соло + 12 парных, охватывают уровни от E (новичок) до A (топ).
- **Словарь из 37 терминов** в 4 категориях: удары, корт, правила и счёт, тактика.
- **Схема корта для каждого drill** — векторная, генерируется на лету из описания в данных.
- **PWA**: можно «установить» на iPhone/Android, работает офлайн после первой загрузки.
- **Mobile-first**, протестировано на ширине от 320px (старый iPhone SE).
- **Светлая тема**, типографика Bricolage Grotesque + DM Sans + JetBrains Mono.
- **Таймер** на длительность упражнения с паузой.
- **Избранное** — сохраняется в localStorage.
- **Случайное упражнение** — кнопка «кубик» в шапке.
- **Нулевые зависимости** — никаких фреймворков и сборщиков. Только HTML/CSS/JS.

## Локальный запуск

Из-за service worker нужен HTTP-сервер (открытие через `file://` сломает PWA-часть, но всё остальное работает).

```bash
# самый простой способ
cd squash-drills
python3 -m http.server 8080
# открыть http://localhost:8080
```

Или `npx serve`, или любой статический сервер.

## Деплой

Вся папка — это и есть проект. Никакой сборки не нужно.

### Cloudflare Pages

1. Зайти на https://dash.cloudflare.com → Pages → Create application → Direct Upload
2. Перетащить папку `squash-drills/` целиком.
3. Готово, получите `https://<имя>.pages.dev`.

### Netlify

1. Зайти на https://app.netlify.com/drop
2. Перетащить папку.
3. Готово.

### Vercel

1. `npm i -g vercel` (или https://vercel.com/new)
2. В папке проекта: `vercel deploy --prod`
3. Готово.

### GitHub Pages

1. Создать публичный репозиторий, залить туда содержимое папки.
2. Settings → Pages → Source: `main` branch, `/` (root).
3. Через минуту работает на `https://<user>.github.io/<repo>`.

## Добавление своих упражнений

Открываем `drills.js`, копируем любую готовую запись и редактируем:

```js
{
  id: 'мой-drill-unique-id',     // уникальный, kebab-case
  name: 'Название',
  players: 1,                     // 1 или 2
  level: 'C',                     // A / B / C / D / E
  difficulty: 3,                  // 1–5 звёзд
  duration: '10 мин',
  focus: ['техника', 'длина'],    // теги
  description: 'Полный текст описания...',
  steps: ['Шаг 1', 'Шаг 2', ...],
  tips: ['Совет 1', 'Совет 2', ...],
  diagram: {
    shots: [
      // удары: from/to — позиция на корте, type — тип удара
      { from: 'BR', to: 'FW_R', type: 'drive', label: '1' },
    ],
    players: [
      { at: 'T', label: 'A' },            // основной игрок (тёмный круг)
      { at: 'BL', label: 'B', role: 'B' }, // партнёр (белый круг)
    ],
    zones: [
      // подсвеченная зона
      { at: [0, 240, 50, 65], label: 'цель' },
    ],
  },
},
```

### Позиции на корте

Можно использовать ключи (см. `court.js`) или координаты `[x, y]` в системе `0..200` × `0..305`:

- `T` — точка T (центр короткой линии)
- `BL` / `BR` — задний левый / правый угол
- `FL` / `FR` — передний левый / правый
- `FW_L` / `FW_C` / `FW_R` — передняя стена (лево / центр / право)
- `SB_L` / `SB_R` — центр левого / правого квадрата подачи
- `SW_L_BACK`, `SW_R_BACK` — середина боковых стен сзади
- и т.д. — все позиции перечислены в `court.js`

### Типы ударов (для цветов и легенды)

`drive`, `cross`, `boast`, `drop`, `lob`, `volley`, `serve`

## Добавление терминов в словарь

Открываем `glossary.js`, копируем любую запись:

```js
{
  id: 'мой-термин',
  category: 'shots',          // 'shots' | 'positions' | 'rules' | 'tactics'
  ru: 'Название',
  en: 'English equivalent',
  def: 'Определение в 1–3 предложениях.',
},
```

Категории и их подписи лежат в массиве `GLOSSARY_CATEGORIES` — туда же можно добавлять новые.
